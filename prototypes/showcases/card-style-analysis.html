🎯 我的真實建議：混合風格策略

不要選單一風格——用「卡牌類別 × 風格」的對應，讓視覺差異本身就傳達遊戲資訊。

⚙️ 機組卡
真實感 + 科技線條

🔧 解決「不真實」的具體方法

為什麼 GPT 生圖會失真？因為它沒看過足夠的真實風電設備圖。葉片內部、齒輪箱、機艙這些工程細節，AI 是用想像力填補的——這就是 F05 卡會「機艙噴火」的原因。

✅ 三個提高真實性的具體做法

方法 1：用 reference image 餵 AI

從 IEA Wind、IEC 標準文件、風機廠商型錄（Vestas、Siemens Gamesa、GE）抓真實設備示意圖，當作 Midjourney/DALL·E 的 --cref 或 reference image。生成時 AI 會模仿這些細節。

方法 2：分層生成（背景 + 設備 + 數據面板）

不要要 AI 「畫一張完整卡片」，而是分開生成：
1. 背景（風電場景）
2. 設備（風機/齒輪箱）— 用 reference
3. 數據面板與標註（自己用 Figma/PowerPoint 疊上）
這樣每一層都可控。

方法 3：工程師 review 制度

每張卡圖生成後，讓你的研究生或業界朋友看一眼「這設備位置對嗎」。AI 生圖最大問題是「看起來像但細節錯」，人類專家 5 秒就能抓到。
可以建立簡單的 review checklist：齒輪箱位置、葉片數量、塔筒比例、感測器位置等。

🤖 改進的 AI Prompt 模板

下次生圖試試這個結構，比直接「畫一張齒輪箱故障」效果好十倍：

[STYLE] Technical engineering schematic illustration, IEC 61400 documentation style
[SUBJECT] Wind turbine gearbox abnormal vibration, planetary gear system cutaway view
[DETAILS] Cross-section showing main shaft, planetary gears, high-speed shaft, bearings; with measurement annotations and vibration sensor placement
[CAMERA] Front-facing technical illustration angle, slight isometric
[REFERENCE] Style of NREL technical reports, Vestas product brochures
[AVOID] ❌ fire, smoke, dramatic flames, anime style, cute characters
[COLOR] Industrial palette: steel grey, signal red for fault indicators, technical white background
[OUTPUT] 4:5 portrait, 300 DPI, sharp focus on mechanical details

關鍵字技巧：
- 加 "technical illustration" 而不是 "illustration"
- 加 "cross-section / cutaway view" 觸發剖視圖
- 加 "NREL style" 或 "IEC documentation style" 拉真實感
- 用 negative prompt（--no 或 [AVOID]）排除戲劇化效果

💎 我的最終建議

選風格 4「工程示意圖」作為主軸，理由：

  1. 真實性 = 教學工具的命根子。你是大學老師，不能讓學生記錯設備樣子。
  2. F08 已經證明這風格可行。三張圖卡裡，工程師會覺得 F08 最不刺眼。
  3. 跟你的研究背景一致。SCADA、CMS、IEC 61400 都是工程文件風格，視覺一致才有品牌感。
  4. 容易製作。AI 用對 prompt 可以一致產出，不用每張都找美術改。
  5. 論文截圖時不會丟臉。「我們開發的教學遊戲」配漫畫風會弱化研究感。

但機組卡保留你 M07 那種 CG 風格——那是遊戲的「英雄卡」，需要視覺衝擊。其他卡走工程示意，整體有層次。

下一步建議：選 3 張「最關鍵」的卡（M07 機組 + T03 技師 + F06 故障），各做 3 版風格測試，找你身邊的人（工程師、學生、家人）投票。資料來決定，不要靠直覺。
